# 周末去哪儿 — 产品设计规格书

> 版本: v1.0 | 日期: 2026-06-14 | 状态: 设计完成

---

## 一、背景与问题

### 痛点
- 用户在抖音、小红书、B站等多个平台刷到好吃好玩的内容并收藏
- 收藏分散在多个平台，周末想找时难以检索
- 缺乏统一的整理、分类、地图查看和分享能力

### 目标
构建一个微信小程序「周末去哪儿」，帮助用户跨平台聚合收藏的美食和游玩地点，周末快速决策。

### 目标用户
- 喜欢吃喝玩乐的年轻人（20-35岁）
- 高频刷社交平台并收藏探店内容
- 周末有外出消费习惯

---

## 二、产品概要

| 维度 | 决策 |
|------|------|
| 产品形态 | 微信小程序 |
| 产品名称 | 周末去哪儿 |
| 视觉风格 | 清新美食风（暖色调、圆润卡片、美食大图） |
| 交付策略 | 分三阶段迭代，每阶段独立可用 |

## 三、分阶段功能规划

### 🥇 第一阶段 — 「收藏管家」（2-4周）

核心闭环：收藏 → 整理 → 查找 → 分享

**功能列表：**
- **添加收藏**：粘贴链接自动解析（标题/封面/平台），AI补全分类标签建议，用户确认保存
- **列表浏览**：瀑布流卡片展示，按分类/标签筛选，关键词搜索
- **地图模式**：腾讯地图展示收藏标记点，区域浏览
- **收藏详情**：地点信息、原始链接、地图定位、一键导航
- **微信分享**：分享单个收藏给微信好友（小程序卡片）
- **个人中心**：收藏统计、分类管理、设置

**页面结构（5个Tab）：**
1. 首页 — 快捷入口 + 最近收藏
2. 地图 — 收藏地点地图视图
3. 添加（居中突出）— 核心操作入口
4. 列表 — 分类筛选 + 搜索
5. 我的 — 统计 + 设置

**关键交互：**
- 添加成功动效（震动 + 星星动画）
- 剪贴板监听（检测链接自动提示添加）
- 长按拖拽排序
- 下拉刷新 + 骨架屏加载

### 🥈 第二阶段 — 「智能助手」（2-3周）

- 语音录入：说出地点信息，AI自动解析保存
- 截图OCR：识别截图中的地点名称和描述
- AI周末推荐：根据天气、距离、历史偏好推荐2-3个地方
- 收藏数据统计面板

### 🥉 第三阶段 — 「社交发现」（2-3周）

- 好友收藏互相可见（需授权）
- 热门地点发现
- 个性化口味推荐优化
- 探店打卡记录
- 美食图谱生成

## 四、技术架构

### 整体分层
```
┌────────────────────────────────────┐
│        微信小程序前端 (原生+WeUI)      │
│   首页 │ 地图 │ 添加 │ 列表 │ 我的    │
├────────────────────────────────────┤
│      微信云开发 CloudBase            │
│  云函数 │ 云数据库 │ 云存储 │ CDN     │
├────────────────────────────────────┤
│        外部服务 (API调用)             │
│  腾讯地图 │ 链接解析 │ AI大模型        │
└────────────────────────────────────┘
```

### 技术选型

| 模块 | 选型 | 说明 |
|------|------|------|
| 前端框架 | 微信原生 + WeUI | 小程序原生开发，轻量快速 |
| 后端服务 | CloudBase 云开发 | 免运维、自带鉴权、按量付费 |
| 数据库 | 云数据库 (NoSQL) | JSON文档存储，灵活扩展 |
| 文件存储 | 云存储 + CDN | 图片自动压缩 + 防盗链 |
| 地图服务 | 腾讯地图 wx. API | 小程序原生地图组件 |
| 链接解析 | 云函数 + HTTP抓取 | 抓取OG标签/标题/描述 |
| AI能力 | 云函数调用大模型API | OCR/语音/推荐（P2） |

## 五、视觉设计方向

### 配色方案（清新美食风）
- 主背景：暖白 `#FFF8F0`
- 主色调：暖橙 `#FF9A56`
- 辅助色：橄榄绿 `#8B9D6B`
- 点缀色：珊瑚红 `#FF6B6B`
- 中性色：米灰 `#E8D5B7`
- 文字：深棕 `#4A3728`

### 设计语言
- 圆润卡片（大圆角 16-20px）
- 美食大图封面
- 柔和阴影
- 温暖友好的品牌调性

---

## 六、非功能需求

- **性能**：首页加载 < 2s，列表滑动流畅 60fps
- **可用性**：核心流程（添加收藏）≤ 3步完成
- **兼容性**：支持微信 8.0+，iOS 14+ / Android 8+
- **安全**：用户数据隔离，云函数鉴权
- **扩展性**：数据模型支持灵活添加字段，不破坏已有数据

---

## 七、验证方案

### P1 验证标准
1. 用户可在 3 步内完成一次收藏添加
2. 地图模式下正确展示所有收藏标记点
3. 列表筛选后结果准确
4. 分享卡片好友可正常打开查看
5. 在不同机型上页面正常渲染

### 后续验证
- P2：AI推荐准确率 > 70% 用户满意度
- P3：社交功能用户参与率 > 30%
